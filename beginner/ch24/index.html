  2. Introducing the DOM

  3. The Document Object

    Everything in the Document is an object and they are connected as a tree I know from Datastructures from my computer science study

  4. getElementById

    You get the DOM-Object. There should always be only one element with a certain ID. Therefore it's get ElementById and getElementsByClassName

  5. getElementsByTagName & className

    console.dir gives a information about objects in the console. GetElementsByTagName and getElementsByClassName don't return arrays. It's a iterable collections. Colt uses for/of loop.

  6. querySelector & querySelectorAll

    CSS-Style query selector. QuerySelector gives only the first object. QuerySelectorAll gives you collection of matching elements.

  7. innerHTML, textContent, & innerText

    textContent gives the content like it's saved in the file. Even hidden content. InnerText will not get you content that's hidden. InnerHTML gives the whole content in HTML obviously.

  8. Attributes

    This topic will be more important when handling with data or when using Reacts

  9. Changing Styles

    While you look up in the DOM and check for example h1.style.color you will find every possible method setting a style, but it will be empty because it only contains inline styles.
    You shouldn't use this method for styling. Better you give elements a class.

  10. ClassList

    One of the most used things. Ex: h2.classList.{add|remove|contains|toggle}

  11. Traversing Parent/Child/Sibling

    Just how you would traverse in a tree data structure. parentElement | children | childElement | firstChild | lastChildElement | nextSibling
    Nodes are DOM objects. They are represented as text, mostly whitespace like return character. childre will give an html collection. Elements are HTML elements, that's mostly used.

  12. Append & AppendChild

    Create element document.createElement('img') then append on an element. There is also prepend and the option to add an element between elements. Element.insertAdjacentElement()

  13. removeChild & remove

    RemoveChild requires to select the parent, then you can remove the child. Nowadays we can use remove and can select the actual element we want to remove
